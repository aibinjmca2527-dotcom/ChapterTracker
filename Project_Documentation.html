1. Project Overview

The **Chapter Tracker** is a highly efficient web application designed to track the collective reading progress of a shared group of readers in real-time. Originally written in Next.js, the codebase has been fully rewritten into a clean **Python Flask** backend and a unified **React** frontend, optimized to run seamlessly on Vercel's cloud infrastructure.

2. Modern Tech Stack

  • Backend: Python Flask (micro-framework serving the frontend and the REST API).
  • Frontend: Modern responsive HTML5/CSS3 coupled with standalone React & Babel (injected in templates/index.html) for clean component architecture and rapid state management without requiring complex Node build tools.
  • Database Layer: High-performance dual-engine database setup:
    • Vercel KV (Redis): Used in production on Vercel for 100% permanent data storage that never resets.
    • SQLite (Local File): Used as an automatic fallback when running locally for zero-config development.

3. System Architecture & Workings

The application implements a smart fallback mechanism inside the database layer (api/index.py) to balance local simplicity and production durability:

Local Development Flow

When running the application locally on your computer, the environment variable detector checks for Vercel credentials. Finding none, it automatically creates and reads from a local SQLite file (database.db). Any changes you make are instantly preserved in this local database file.

Production/Vercel Flow

When deployed on Vercel and connected to **Vercel KV/Upstash Redis**, the application detects the standard KV_REST_API_URL and KV_REST_API_TOKEN environment variables. It bypasses SQLite and performs super-fast RESTful HTTP requests using Python's built-in urllib.request library to load/save JSON data directly in the cloud. This makes the database 100% persistent across sleep intervals.

4. Core API Endpoints

The backend serves three primary HTTP routes:

1. Home Page Route

GET /

Serves the clean React client UI in templates/index.html.

2. Login Endpoint

POST /api/login
Body: { "email": "aibin@gmail.com", "password": "aibin123" }

Verifies reader email and password credentials and grants access.

3. Chapters Endpoint

GET /api/chapters
Response: { "Aibin": [1, 2], "Milan": [3, 4], ... }

POST /api/chapters
Body: { "name": "Aibin", "chapters": [1, 2, 3] }

Reads the collective progress or registers newly completed chapters for a specific reader.

5. Excel & CSV Reporting

The frontend client has a built-in "Download Excel Report" button. Clicking this triggers a browser script that builds a clean comma-separated table in the following structure:

"Aibin","Milan","Edwin","Seba","Lakshmi"
"Chap 1","Chap 3","","",""
"Chap 2","Chap 4","","",""

This triggers an automatic download of a chapter_tracker.csv spreadsheet which can be double-clicked to view or print immediately in Microsoft Excel or Google Sheets.

6. Quick Setup & Future Restarts

Running Locally

  1. Ensure Python is installed on your computer.
  2. Open a command prompt in the C:\chappp folder and run: 
    pip install -r requirements.txt
python api/index.py
  3. Open your browser to http://127.0.0.1:5000 to view and test your application.

Updating Git & Vercel

Whenever you modify files in the future, run these commands to update your live website:

git add .
git commit -m "Update chapter tracker project"
git push origin main

Vercel will build and publish the changes within 30 seconds automatically.
